# Environment Matrix (child)

A consolidated matrix across DEV, QA, PERF, DEMO using and extending:

- FinX Environments
- Versioning Standards v1.2

Per-environment:

- Purpose, audience, and required capabilities.
- Typical activities and which test types run where.
- Data strategy (synthetic vs vendor sandboxes, constraints on PII).
- Access controls and stability expectations.

Promotion flow DEV -> QA -> PERF -> DEMO aligned with versioning and quality gates (DEMO = production-equivalent, CAB-style discipline).
